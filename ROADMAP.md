# ROADMAP — StroyDocs

> Дорожная карта разработки. Обновляется по мере прохождения фаз.
> Статусы: ⬜ не начато · 🔄 в работе · ✅ готово · ⏸ отложено

## Фаза 1 — Ядро и фундамент
**Цель:** Рабочее приложение с авторизацией, организациями и проектами.
**Ориентир:** 2–3 недели

### Инфраструктура
- ⬜ Инициализация Next.js 14 (App Router, TypeScript, Tailwind CSS)
- ⬜ Подключение shadcn/ui + настройка темы (синий #2563EB)
- ⬜ Настройка Prisma + docker-compose (PostgreSQL + Redis + MinIO)
- ⬜ Базовая структура папок согласно CLAUDE.md
- ⬜ .env.example с переменными Timeweb Cloud
- ⬜ ESLint + Prettier + TypeScript strict mode

### Авторизация (Auth)
- ⬜ Регистрация организации (название, ИНН, email, пароль)
- ⬜ Вход / выход (NextAuth.js + JWT)
- ⬜ Защищённые маршруты (middleware)
- ⬜ Страница приглашения сотрудника по email-ссылке
- ⬜ Роли: `admin | manager | worker | controller | customer`

### Layout и навигация
- ⬜ Фиксированный левый sidebar
  - ⬜ Логотип StroyDocs
  - ⬜ Блок профиля (аватар, ФИО, роль)
  - ⬜ Иконка уведомлений со счётчиком
  - ⬜ Список проектов пользователя
  - ⬜ Название компании + ссылки (справка, обратная связь)
- ⬜ Базовый dashboard (главная страница после входа)
- ⬜ Вкладочная структура (Tabs) внутри разделов

### Организация
- ⬜ Карточка компании (ИНН, ОГРН, данные СРО)
- ⬜ Реестр сотрудников (таблица: ФИО, должность, статус, email, телефон)
- ⬜ Кнопка "Нанять сотрудника" → модальное окно + отправка приглашения

### Проекты
- ⬜ Список проектов (таблица с фильтром и поиском)
- ⬜ Создание проекта → модальное окно
  - ⬜ Поля: название, адрес, генподрядчик, заказчик
  - ⬜ Автозаполнение по ИНН через API ФНС/ЕГРЮЛ
- ⬜ Карточка проекта (вкладки: Договоры, Статистика, Документы)

### Договоры
- ⬜ Список договоров проекта
- ⬜ Создание договора → иерархия Проект → Договор → Субдоговор
- ⬜ Карточка договора:
  - ⬜ Участники (застройщик, подрядчик, авторнадзор, субподрядчик)
  - ⬜ Приказы о назначении ответственных лиц

### База данных (Prisma schema — Фаза 1)
- ⬜ `Organization` (ИНН, ОГРН, СРО, реквизиты)
- ⬜ `User` (ФИО, email, телефон, роль, organizationId)
- ⬜ `Invitation` (email, токен, роль, статус)
- ⬜ `Project` (название, адрес, organizationId)
- ⬜ `Contract` (тип, участники, parentId для субдоговоров)
- ⬜ Миграции + `prisma db seed` (тестовые данные)

## Фаза 2 — Производство работ
**Цель:** Учёт работ, материалов и сертификатов.
**Ориентир:** 2–3 недели

### КСИ (Классификатор строительной информации)
- ⬜ Загрузка JSON-справочника КСИ в БД (seed)
- ⬜ Модель `KsiNode` (id, parentId, code, name)
- ⬜ Компонент выбора из многоуровневого дерева КСИ
- ⬜ Привязка видов работ к узлам КСИ

### Материалы
- ⬜ Реестр материалов по договору (таблица)
- ⬜ Добавление материала → модальное окно
  - ⬜ Поля: наименование, поставщик, номер накладной, количество, единица
- ⬜ Учёт остатков (получено / использовано / остаток)
- ⬜ ⚠️ Автоматическое предупреждение при отсутствии сертификата качества
- ⬜ Загрузка сопроводительных документов (паспорта, сертификаты) → Timeweb S3
- ⬜ Встроенный просмотр PDF прямо в интерфейсе

### Записи о выполненных работах
- ⬜ Реестр выполненных работ (таблица по договору)
- ⬜ Создание записи о работе:
  - ⬜ Выбор шифра проекта
  - ⬜ Место проведения работ
  - ⬜ Выбор вида работ (дерево КСИ)
  - ⬜ Привязка материалов + списание из остатков
  - ⬜ Выбор норматива (ГОСТ, СП, СНиП)
- ⬜ Цветовая индикация статусов

### База данных (Фаза 2)
- ⬜ `KsiNode` (дерево КСИ)
- ⬜ `WorkItem` (вид работ: ksiNodeId, contractId, шифр)
- ⬜ `Material` (накладная, количество, остаток, contractId)
- ⬜ `MaterialDocument` (s3Key, тип: паспорт/сертификат)
- ⬜ `WorkRecord` (запись о работе, место, дата)
- ⬜ `MaterialWriteoff` (списание: workRecordId, materialId, количество)

## Фаза 3 — Исполнительная документация
**Цель:** Генерация и подписание ИД, журнал замечаний.
**Ориентир:** 2–3 недели

### Реестры ИД
- ⬜ Список ИД по договору (таблица с фильтрами по типу и статусу)
- ⬜ Типы документов: АОСР, ОЖР, Акт технической готовности

### Генерация PDF
- ⬜ HTML-шаблоны актов в `/templates` (Handlebars)
  - ⬜ Шаблон АОСР (Акт освидетельствования скрытых работ)
  - ⬜ Шаблон ОЖР (Общий журнал работ)
  - ⬜ Шаблон Акта технической готовности
- ⬜ Генерация PDF через Puppeteer (BullMQ-задача)
- ⬜ Сохранение PDF в Timeweb S3
- ⬜ Pre-signed URL для безопасного скачивания

### PDF-просмотрщик
- ⬜ Встроенный многостраничный просмотр (react-pdf)
- ⬜ Навигация по страницам, масштаб
- ⬜ Просмотр исполнительных схем (чертежей)

### Документарий (файловый архив)
- ⬜ Категории: Разрешительная документация, Рабочий проект, Исполнительные схемы, Сертификаты, Нормативка
- ⬜ Загрузка файлов → Timeweb S3
- ⬜ Штамп "Копия верна" (наложение на PDF с ФИО и должностью)
- ⬜ Связь листов, шифров и дат выдачи для рабочего проекта

### Журнал замечаний
- ⬜ Добавление замечания к конкретному месту в документе
- ⬜ Статус замечания: открыто / устранено (переключатель)
- ⬜ История изменений статуса

### База данных (Фаза 3)
- ⬜ `ExecutionDoc` (тип, статус, contractId, s3Key, generatedAt)
- ⬜ `Signature` (userId, docId, тип подписи, timestamp, s3Key подписи)
- ⬜ `DocComment` (executionDocId, текст, координаты на странице, статус)
- ⬜ `Archive` (категория, s3Key, sheetNumber, cipher, issueDate)

## Фаза 4 — ЭЦП и интеграции
**Цель:** Электронная подпись и подключение внешних сервисов.
**Ориентир:** 2 недели

### Электронная цифровая подпись (ЭЦП)
- ⬜ Абстракция `SignatureProvider` (интерфейс для разных провайдеров)
- ⬜ Интеграция КриптоПро CSP (через REST API шлюз)
- ⬜ Режимы подписания:
  - ⬜ Открепленная подпись (detached)
  - ⬜ Встроенная подпись (embedded)
- ⬜ Проверка подписи при открытии документа
- ⬜ Хранение подписей в Timeweb S3

### Интеграция с API ФНС / ЕГРЮЛ
- ⬜ Автозаполнение реквизитов организации по ИНН
- ⬜ Автозаполнение данных о контрагентах при создании договора
- ⬜ Кэширование результатов запросов (Redis, TTL 24ч)

### Внешние интеграции (настройка шины данных)
- ⬜ U-lab
- ⬜ EXON
- ⬜ Cynteka
- ⬜ Основа
- ⬜ UI управления интеграциями в разделе "Организация"

### Уведомления
- ⬜ Лента событий (назначение ролей, изменение статусов ИД)
- ⬜ Email-уведомления (BullMQ + SMTP)
- ⬜ In-app уведомления с счётчиком в sidebar
